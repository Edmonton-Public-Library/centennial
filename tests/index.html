Acceptance Test Documentation

Introduction

The purpose of this documentation is to provide a series of acceptance tests to ensure that the product is in an acceptable state for general distribution.

This document is intended for all those who will be verifying the integrity of the overall functionality of the product.

Installation

  1. Untar the project into the desired directory.
  2. In a terminal run the INSTALL script.

System Tests

Below is the list of system tests. Tests are split into the TimeMap Tests and Hundred Year Quest Tests.

Administrative


# TM-Test.1.1
Test Name Approve story
Purpose The purpose of this test is to ensure content is not displayed until approved by an administrator
User Requirement Reference TM.1.1
Pre Conditions Story must be uploaded but unapproved
Post Conditions Story must be visible on public page
Test Steps
Step # Description Expected Result
1 Navigate to administrator page Content should be available on page
2 Verify Content is appropriate for display Pass
3 Approve content Pass
4 Verify that content is now visible on public webpage Content should be visible
5 Return to administrator page and delete the approved content Pass
6 Verify that the story is no longer available on the TimeMap page. Pass


# TM-Test.1.2
Test Name Reject Story
Purpose Ensure that unapproved content is never displayed on the public webpage and is deleted.
User Requirement Reference TM.1.1
Pre Conditions Story must be uploaded but unapproved.
Post Conditions Story must be deleted and not visible to public.
Test Steps
Step # Description Expected Result
1 Navigate to administrator page Content should be available on page
2 Verify Content is inappropriate for display Pass
3 Reject the uploaded content. Pass
4 Verify that the content was deleted. Pass
5 Verify that the content is not visible on the public webpage Pass


# TM-Test.1.3
Test Name Ban User
Purpose Test to see if banning a user prevents them from logging on
User Requirement Reference TM.1.4
Pre Conditions User account must already exist
Post Conditions User should no longer exist and should not be able to logon or recreate his account
Test Steps
Step # Description Expected Result
1 Navigate to account management administrator page. Pass.
2 Locate the user to delete and click delete button. Confirmation to delete should appear
3 Delete the user Confirmation of user deletion.
4 Navigate to main TimeMap page Pass
5 Attempt to login with deleted user Login should fail
6 Attempt to re-create account with same credentials Attempt should fail
